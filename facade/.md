# Padrão Facade

## Introdução
O padrão Facade fornece uma interface unificada para um conjunto de interfaces em um subsistema.

## Por que foi criado
Foi criado para simplificar o uso de subsistemas complexos, escondendo detalhes internos.

## Quando usar
Use quando um subsistema tem muitas partes e você quer uma interface simples para o cliente.

## Comparação com outros métodos
Diferente de expor tudo diretamente, o Facade cria uma camada de abstração.

## Prós e Contras
**Prós:** Simplifica uso, reduz dependências, fácil de manter.  
**Contras:** Pode esconder funcionalidades importantes se mal usado.
