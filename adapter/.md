# Padrão Adapter

## Introdução
O padrão Adapter permite que classes com interfaces incompatíveis trabalhem juntas. Ele adapta uma interface para outra que o cliente espera.

## Por que foi criado
Foi criado para reutilizar classes existentes sem mudar o código delas, quando as interfaces não combinam.

## Quando usar
Use quando você tem uma classe antiga que não funciona com uma nova interface, e quer conectar elas sem mexer no código antigo.

## Comparação com outros métodos
Diferente de herança, que cria subclasses, o Adapter usa composição para conectar interfaces diferentes. É mais flexível que mudar o código diretamente.

## Prós e Contras
**Prós:** Reutiliza código antigo, fácil de adicionar.  
**Contras:** Pode adicionar complexidade se tiver muitas adaptações.
