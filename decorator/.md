# Padrão Decorator

## Introdução
O padrão Decorator adiciona responsabilidades a objetos dinamicamente, sem alterar a classe.

## Por que foi criado
Foi criado para estender funcionalidades de objetos de forma flexível, evitando subclasses para cada combinação.

## Quando usar
Use quando quer adicionar comportamentos extras a objetos em tempo de execução, como bordas em janelas.

## Comparação com outros métodos
Diferente de herança, que é estática, o Decorator permite composições dinâmicas.

## Prós e Contras
**Prós:** Flexível, segue Open-Closed, fácil de combinar.  
**Contras:** Pode criar muitos objetos pequenos, difícil de debugar.
