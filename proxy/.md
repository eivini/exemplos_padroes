# Padrão Proxy

## Introdução
O padrão Proxy fornece um substituto para controlar o acesso a outro objeto.

## Por que foi criado
Foi criado para controlar acesso a objetos caros ou sensíveis, como lazy loading ou segurança.

## Quando usar
Use quando quer adicionar controle, como carregar objetos só quando necessário.

## Comparação com outros métodos
Diferente do Decorator, que adiciona responsabilidades, o Proxy controla acesso.

## Prós e Contras
**Prós:** Adiciona funcionalidades sem mudar objeto, controla acesso.  
**Contras:** Pode adicionar latência, aumenta complexidade.

## Aplicações Reais
Em bancos, proxies são usados para controlar acesso a sistemas internos, como lazy loading de dados ou segurança. Diferente de VPN, que criptografa toda a conexão para acesso remoto seguro, o proxy age como intermediário para requisições específicas, sem criptografar tudo. Bancos usam VPNs para funcionários acessarem redes seguras de fora, enquanto proxies filtram tráfego interno.
